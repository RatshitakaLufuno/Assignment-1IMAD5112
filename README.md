1. Purpose of the App
 
The Social Spark mobile application was developed to help users maintain meaningful social connections during busy daily routines. Many individuals, including the target user “Cora,” struggle to stay in touch with friends, family, and colleagues due to time constraints.

This app solves that problem by:
 • Providing quick, actionable social suggestions
 • Tailoring suggestions based on the time of day
 • Encouraging consistent communication habits

The application promotes small but impactful actions such as sending a message, making a quick call, or engaging on social media.


2. The app’s objectives

The Social Spark app's primary goals are:
• To make social interactions easier by utilizing time-based recommendations
• To offer an intuitive and captivating user interface
• To use conditional logic (if/when statements); 
• To illustrate the application of Kotlin programming concepts
• To implement input validation and error handling
• To incorporate GitHub and GitHub Actions for automation and version control
3. Design Considerations
Usability, simplicity, and aesthetic appeal were given top priority during the app's design.

3.1 Design of User Interfaces

The user interface consists of:
• A text input field where the time of day can be entered
• A "Get Social Spark" button to produce recommendations
• A TextView to show outcomes. 
• A Reset button to remove inputs

 3.2 Visual Design Decisions 
• Bold headlines enhance readability 
• A gentle purple/lavender background was employed to create a serene and welcoming atmosphere
• Buttons are easy to see and use.
• Layout and spacing guarantee a neat and uncluttered user interface.

3.3 User Experience (UX)
The app guarantees: Simple navigation with few steps; instant feedback following user input; unambiguous error messages when incorrect input is entered; and quick interaction without needless complication


4. Handling Errors

Error handling was put in place to enhance usability:
• The user is prompted to provide a value if the input field is empty; if incorrect input is supplied, a helpful message is shown.
• To prevent case sensitivity problems, input is cut and changed to lowercase.

This guarantees a seamless and trouble-free user experience.

5. Reset Features

The button for reset:
• The input field is cleared.
• Removes the suggestion that is displayed
• Shows a message of confirmation

This feature enhances usability and makes it simple for the user to resume interactions.
